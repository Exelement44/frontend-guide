Введение в таблицы и формы

Теперь, когда вы освоили базовые элементы контента, пришло время изучить два мощных инструмента HTML: таблицы для структурированного представления данных и формы для взаимодействия с пользователем. Без этих элементов современный веб был бы невозможен.

Создание таблиц для данных

Таблицы в HTML используются не для создания макета страницы (для этого есть CSS), а исключительно для отображения табличных данных: расписаний, прайс-листов, сравнительных характеристик.

Базовая структура таблицы:

Таблица строится из нескольких обязательных тегов, которые работают как конструктор:

  1. <table> - главный контейнер для всей таблицы.
  2. <tr> (table row) - строка таблицы.
  3. <td> (table data) - ячейка с данными внутри строки.
  4. <th> (table header) - ячейка-заголовок для столбца или строки. Текст внутри автоматически выделяется жирным и центрируется.
<table>
    <tr>
        <th>Месяц</th>
        <th>Доход</th>
    </tr>
    <tr>
        <td>Январь</td>
        <td>100 000 ₽</td>
    </tr>
    <tr>
        <td>Февраль</td>
        <td>120 000 ₽</td>
    </tr>
</table>

Улучшение читаемости:

Для лучшей структуры и доступности используются дополнительные теги:

  • <thead> - оборачивает блок с заголовками (<tr> с <th>).
  • <tbody> - оборачивает основное тело таблицы с данными.
  • <tfoot> - оборачивает итоговую строку (например, «Всего»).
<table>
    <thead>
        <tr><th>Продукт</th><th>Цена</th><th>Кол-во</th></tr>
    </thead>
    <tbody>
        <tr><td>Хлеб</td><td>50 ₽</td><td>1</td></tr>
        <tr><td>Молоко</td><td>90 ₽</td><td>2</td></tr>
    </tbody>
    <tfoot>
        <tr><td colspan="2">Итого:</td><td>230 ₽</td></tr>
    </tfoot>
</table>

ВАЖНО: colspan="2" в примере выше объединяет две ячейки по горизонтали. Аналогично rowspan объединяет ячейки по вертикали.

Элементы форм (input, textarea, button)

Формы - это инструмент диалога с пользователем. Они собирают данные для поиска, регистрации, отправки комментариев, оформления заказов.

Базовая структура формы:

Любая форма начинается с тега <form>. У него есть ключевые атрибуты:

  • action - URL, куда отправятся собранные данные.
  • method - метод отправки (обычно "get" или "post").
<form action="/send-data" method="post">
    <!-- Сюда вставляются поля формы -->
</form>

Поля для ввода (тег <input>):

Это самый универсальный и часто используемый элемент форм. Его поведение кардинально меняется в зависимости от атрибута type.

<label for="name">Ваше имя:</label>
<input type="text" id="name" name="user_name" placeholder="Иван Иванов">

<label for="email">E-mail:</label>
<input type="email" id="email" name="user_email">

<label for="psw">Пароль:</label>
<input type="password" id="psw" name="user_psw">

<!-- Группа переключателей (выбор одного) -->
<p>Выберите способ связи:</p>
<input type="radio" id="contact_phone" name="contact" value="phone">
<label for="contact_phone">Телефон</label>
<input type="radio" id="contact_email" name="contact" value="email" checked>
<label for="contact_email">E-mail</label>

<!-- Флажки (можно выбрать несколько) -->
<input type="checkbox" id="subscribe" name="subscribe" checked>
<label for="subscribe">Подписаться на новости</label>

ТАКЖЕ ВАЖНО: Атрибут name - это имя поля, под которым данные уйдут на сервер. Атрибут id связывает поле с тегом <label>, что улучшает доступность (клик по label активирует поле).

Многострочный текст (тег <textarea> ):

Используется для ввода больших текстов: комментариев, сообщений, описаний.

<label for="message">Ваше сообщение:</label>
<textarea id="message" name="user_message" rows="5" cols="30" placeholder="Напишите здесь..."></textarea>

Кнопки (теги <button> и <input type="submit">):

Отправляют форму или выполняют другие действия.

<!-- Основная кнопка отправки формы -->
<button type="submit">Отправить данные</button>
<!-- или -->
<input type="submit" value="Отправить">

<!-- Кнопка для сброса формы -->
<button type="reset">Очистить форму</button>

Простая валидация

Валидация — это проверка введенных данных на корректность. Самую простую валидацию можно настроить прямо в HTML с помощью атрибутов тега <input>.

  • Обязательное поле (required): Браузер не отправит форму, пока это поле не будет заполнено.
  • Проверка формата через type: Браузер проверит, соответствует ли ввод указанному типу.
  • Паттерн (pattern): Проверка по регулярному выражению. Например, для российского номера телефона.
<input type="tel" name="phone" pattern="[\+]\d{1}\s[\(]\d{3}[\)]\s\d{3}[\-]\d{2}[\-]\d{2}" placeholder="+7 (999) 123-45-67" title="Формат: +7 (999) 123-45-67">

Если валидация не пройдена, браузер покажет подсказку из атрибута title и не отправит форму.

Практика: Создаем страницу регистрации и таблицы

Давайте создадим страницу, которая объединяет таблицу данных и форму для регистрации на мероприятие.

  1. Создайте файл event.html и подготовьте базовую структуру HTML-документа.
  2. Добавьте таблицу с расписанием докладчиков в тело страницы (<body>):
    3. <h2>Расписание конференции</h2>
<table>
    <thead>
        <tr>
            <th>Время</th>
            <th>Докладчик</th>
            <th>Тема</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>10:00 - 11:00</td>
            <td>Анна Петрова</td>
            <td>Будущее веб-разработки</td>
        </tr>
        <tr>
            <td>11:15 - 12:15</td>
            <td>Игорь Сидоров</td>
            <td>Введение в искусственный интеллект</td>
        </tr>
        <tr>
            <td>13:30 - 15:00</td>
            <td colspan="2">Круглый стол: Обсуждение трендов</td>
        </tr>
    </tbody>
</table>
  3. Под таблицей создайте форму регистрации:
    4. <h2>Регистрация на конференцию</h2>
<form action="#" method="post">
    <label for="reg_name">ФИО *</label><br>
    <input type="text" id="reg_name" name="name" required placeholder="Петров Алексей"><br><br>

    <label for="reg_email">E-mail *</label><br>
    <input type="email" id="reg_email" name="email" required placeholder="alexey@example.com"><br><br>

    <label>Выберите тариф:</label><br>
    <input type="radio" id="tariff_standard" name="tariff" value="standard" checked>
    <label for="tariff_standard">Стандарт (бесплатно)</label><br>
    <input type="radio" id="tariff_vip" name="tariff" value="vip">
    <label for="tariff_vip">VIP (+ доступ к материалам)</label><br><br>

    <label for="reg_comment">Комментарий или пожелания:</label><br>
    <textarea id="reg_comment" name="comment" rows="3" cols="40"></textarea><br><br>

    <input type="checkbox" id="reg_agreement" name="agreement" required>
    <label for="reg_agreement">Я согласен на обработку персональных данных *</label><br><br>

    <button type="submit">Зарегистрироваться</button>
    <button type="reset">Очистить форму</button>
</form>
  4. Откройте файл в браузере. Попробуйте отправить форму, не заполнив обязательные поля (отмеченные *) или указав некорректный email. Браузер покажет ошибки валидации. Протестируйте работу радио-кнопок, флажка и кнопок отправки/сброса.
  5. Экспериментируйте:
    • Добавьте в таблицу новую строку с докладчиком.
    • Создайте поле для ввода телефона с простой валидацией pattern="[0-9]{11}".
    • Добавьте в форму поле для выбора даты (<input type="date">).

Что вы узнали

  1. Таблицы создаются с помощью тегов <table>, <tr>, <td>, <th>. Для структуры используются <thead>, <tbody>, <tfoot>.
  2. Ячейки можно объединять атрибутами colspan и rowspan.
  3. Формы (<form>) — это инструмент сбора данных. Их основные элементы: разнотипные поля <input>, многострочное поле <textarea> и кнопки <button>.
  4. Ключевые атрибуты полей: type определяет вид поля, name - имя для отправки данных, id - связь с <label>, required - обязательность заполнения.
  5. Простую проверку (валидацию) данных можно реализовать на уровне HTML с помощью атрибутов required, type="email/url/number" и pattern.
