Что такое блочная модель?

Каждый HTML-элемент в CSS представляет собой прямоугольный блок (box). Блочная модель описывает, как складываются ширина и высота этого блока из нескольких составляющих.

Почему так важна блочная модель?

Представьте, что вы архитектор, который проектирует здание. У вас есть кирпичи (HTML-элементы), но без правил их укладки невозможно построить прочное и красивое здание. Блочная модель — это как раз те правила, которые определяют, как располагаются и взаимодействуют друг с другом все элементы на веб-странице.

Простая аналогия из реальной жизни:

Представьте картину в рамке под стеклом, висящую на стене:

  • Содержимое (content) — сама картина, то, что художник нарисовал
  • Внутренний отступ (padding) — паспарту, белая или цветная полоса между картиной и рамкой
  • Рамка (border) — сама деревянная или пластиковая рамка картины
  • Внешний отступ (margin) — расстояние от этой картины до соседних картин или до края стены

Без этих составляющих картина не будет смотреться гармонично — она либо "утонет" в раме, либо будет выглядеть одиноко на пустой стене.

Составляющие блочной модели (от центра к краям):

  1. Content (Содержимое) — область, где находится текст, изображение или другой контент элемента
  2. Padding (Внутренний отступ) — прозрачная область вокруг содержимого, внутри границы
  3. Border (Граница) — рамка вокруг padding и content
  4. Margin (Внешний отступ) — прозрачная область снаружи границы, отделяющая элемент от соседей

Как браузер рассчитывает размер элемента?

По умолчанию используется стандартная блочная модель. В этой модели свойство width задает только ширину содержимого, а padding, border и margin добавляются к этой ширине.

Давайте посчитаем на примере: если мы зададим элементу ширину содержимого 300 пикселей, внутренние отступы по 20 пикселей с каждой стороны, границу толщиной 5 пикселей и внешние отступы по 30 пикселей, то:

Общая ширина, которую элемент займет на странице = 300 (ширина контента) + 20+20 (левый и правый внутренние отступы) + 5+5 (левая и правая границы) + 30+30 (левый и правый внешние отступы) = 410 пикселей.

Точно так же рассчитывается и высота: высота контента плюс верхние и нижние отступы, границы и внешние отступы.

Практический пример

Давайте создадим два одинаковых по размеру блока, но с разными подходами к расчету размеров:

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Блочная модель в CSS</title>
    <style>
        .box {
            width: 300px;
            height: 150px;
            padding: 20px;
            border: 5px solid #3498db;
            margin: 30px;
            background-color: #f1c40f;
        }
        
        .box-border-box {
            box-sizing: border-box;
            width: 300px;
            height: 150px;
            padding: 20px;
            border: 5px solid #e74c3c;
            margin: 30px;
            background-color: #2ecc71;
        }
    </style>
</head>
<body>
    <div class="box">
        <p>Стандартная модель (content-box)</p>
        <p>Фактическая ширина: 300 + 20+20 + 5+5 = 350px</p>
    </div>
    
    <div class="box-border-box">
        <p>Альтернативная модель (border-box)</p>
        <p>Фактическая ширина: 300px (включая padding и border)</p>
    </div>
</body>
</html>

В первом блоке используется стандартная модель, где свойство width задает только ширину содержимого. Во втором блоке применяется альтернативная модель с помощью свойства box-sizing: border-box, где свойство width задает общую ширину блока, включая внутренние отступы и границы.

Свойства блочной модели: изучаем детально

1. Содержимое (Content) — свойства width и height

Свойства width и height задают размеры области содержимого элемента. Вы можете использовать:

  • Фиксированные значения в пикселях (например, width: 200px)
  • Относительные значения в процентах от размера родительского элемента (например, width: 50%)
  • Автоматическое определение размера (width: auto или height: auto), когда браузер сам рассчитывает размер на основе содержимого

Также существуют дополнительные свойства для ограничения размеров:

  • min-width и max-width — минимальная и максимальная ширина
  • min-height и max-height — минимальная и максимальная высота

2. Внутренние отступы (Padding)

Свойство padding создает пространство между содержимым элемента и его границей. Есть несколько способов задать внутренние отступы:

  • Одно значение — отступы со всех четырех сторон одинаковые (padding: 20px)
  • Два значения — первое для верхнего и нижнего отступов, второе для левого и правого (padding: 10px 20px)
  • Три значения — верхний, горизонтальные (левый и правый), нижний (padding: 10px 20px 15px)
  • Четыре значения — верхний, правый, нижний, левый (по часовой стрелке, padding: 10px 20px 15px 5px)

Можно также задавать отступы для каждой стороны отдельно: padding-top, padding-right, padding-bottom, padding-left.

3. Границы (Border)

Свойства границы определяют рамку вокруг элемента. Основные свойства:

  • border-width — толщина границы (в пикселях или других единицах измерения)
  • border-style — стиль границы (сплошная, пунктирная, точечная и т.д.)
  • border-color — цвет границы

Чаще всего используется сокращенная запись: border: 2px solid #333;, где указывается толщина, стиль и цвет через пробел.

Стили границ (border-style):

  • solid — сплошная линия
  • dotted — точечная граница
  • dashed — пунктирная граница
  • double — двойная линия
  • groove, ridge, inset, outset — объемные эффекты
  • none — нет границы
  • hidden — скрытая граница (как none, но влияет на расчеты)

Можно задавать границы для отдельных сторон: border-top, border-right, border-bottom, border-left.

Скругление углов — свойство border-radius позволяет создавать закругленные углы. Можно задавать одно значение для всех углов или разные значения для каждого угла.

4. Внешние отступы (Margin)

Свойство margin создает пространство за пределами границы элемента, отделяя его от соседних элементов. Синтаксис аналогичен padding:

  • margin: 20px — отступы со всех сторон
  • margin: 10px 20px — вертикальные и горизонтальные отступы
  • margin: 10px 20px 15px 5px — верхний, правый, нижний, левый

Особенности внешних отступов:

  • Можно использовать отрицательные значения (margin-left: -10px)
  • Автоматические отступы (margin: 0 auto) часто используются для центрирования блочных элементов
  • Вертикальные отступы соседних элементов могут "схлопываться" (об этом подробнее ниже)

Ключевое свойство box-sizing

Свойство box-sizing определяет, как браузер рассчитывает ширину и высоту элемента.

Значение content-box (стандартное, по умолчанию):

  • Свойства width и height задают размер только содержимого
  • Внутренние отступы и границы добавляются к общему размеру

Значение border-box (альтернативное, часто более удобное):

  • Свойства width и height задают общий размер элемента, включая внутренние отступы и границы
  • Размер содержимого автоматически уменьшается, чтобы вместить отступы и границы

Многие разработчики начинают свои стили со сброса этого свойства для всех элементов:

* {
    box-sizing: border-box;
}

Это упрощает расчеты при верстке.

Блочные и строчные элементы (для напоминания)

Блочные элементы:

  • Занимают всю доступную ширину
  • Начинаются с новой строки
  • Можно задавать ширину, высоту, отступы и границы

Примеры: <div>, <h1>-<h6>, <p>, <section>, <article>

Строчные элементы:

  • Занимают только необходимую ширину под контент
  • Не начинаются с новой строки, располагаются друг за другом
  • Нельзя задавать ширину, высоту, вертикальные внешние отступы

Примеры: <span>, <a>, <strong>, <em>, <img>

Строчно-блочные элементы (inline-block):

Компромиссный вариант — свойство display: inline-block:

  • Элементы ведут себя как строчные (не начинаются с новой строки)
  • Но можно задавать ширину, высоту, отступы и границы как у блочных

Схлопывание внешних отступов

Важное явление в CSS: Когда два блочных элемента расположены друг под другом, их вертикальные внешние отступы не складываются, а "схлопываются" — берется большее из двух значений.

Например, если у верхнего элемента margin-bottom: 30px, а у нижнего margin-top: 20px, то расстояние между ними будет не 50 пикселей, а 30 пикселей (большее значение).

Когда отступы не схлопываются:

  • У элементов с display: flex или display: grid
  • Когда между элементами есть граница или внутренние отступы
  • Когда элемент имеет position: absolute или position: fixed
  • Когда у родительского элемента задано overflow: auto или overflow: hidden

Частые проблемы и их решения

1. Элемент "вылазит" за пределы родителя

Если содержимое элемента не помещается в заданные размеры, оно может выходить за границы родительского контейнера. Решение — свойство overflow:

.parent {
    overflow: hidden; /* обрезает всё, что выходит за границы */
    /* или */
    overflow: auto; /* добавляет прокрутку при необходимости */
}

2. Центрирование блочного элемента по горизонтали

Для центрирования блочного элемента с фиксированной или процентной шириной:

.block {
    width: 80%; /* или фиксированная ширина, например 500px */
    margin: 0 auto; /* автоматические отступы слева и справа */
}

3. Вертикальное центрирование (простой способ)

С появлением Flexbox вертикальное центрирование стало намного проще:

.parent {
    display: flex;
    align-items: center; /* вертикальное выравнивание */
    justify-content: center; /* горизонтальное выравнивание */
}

4. Учет внешних отступов при использовании float

Когда элементы плавают (свойство float), их родительский контейнер может "схлопываться". Решение — clearfix:

.clearfix::after {
    content: "";
    display: table;
    clear: both;
}

Что вы узнали:

  1. Блочная модель состоит из четырех слоев: содержимое, внутренние отступы, граница и внешние отступы
  2. По умолчанию используется box-sizing: content-box, где ширина и высота задают только размер содержимого
  3. box-sizing: border-box включает внутренние отступы и границы в указанные ширину и высоту
  4. Padding — внутренние отступы между содержимым и границей, margin — внешние отступы между границей и соседними элементами
  5. Вертикальные внешние отступы соседних элементов могут схлопываться (браузер берет большее значение)
  6. Блочные и строчные элементы ведут себя по-разному относительно блочной модели
  7. Использование универсального правила * { box-sizing: border-box; } упрощает верстку и расчет размеров
