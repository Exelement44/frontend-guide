Что такое семантика в HTML5?

Семантика - это смысл, значение. В контексте HTML семантические теги — это теги, которые называют вещи своими именами: «вот шапка сайта», «вот основная навигация», «вот независимая статья», «вот подвал».

Простая аналогия: Организация библиотеки.

Представьте, что HTML-страница — это библиотека:

  • Старый подход (только <div>) - все книги свалены в одну большую кучу. Чтобы найти учебник, нужно перерыть всю стопку.
  • Семантический подход (HTML5) — книги расставлены по полкам с четкими табличками: «Художественная литература», «Научные труды», «Справочники», «Детские книги». Все структурировано и понятно.

Почему это важно? Семантические теги делают ваш код понятным не только для людей, но и для:

  1. Поисковых систем (SEO) - они лучше понимают, какой контент важнее
  2. Скринридеров (для слабовидящих) - программы могут правильно «озвучить» структуру страницы
  3. Браузеров - они могут предлагать дополнительные функции для определенных типов контента
  4. Будущих разработчиков - код читается как книга

Базовая структура семантической страницы

Вот как выглядит современный каркас типичной веб-страницы с использованием HTML5:

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Мой блог о технологиях</title>
</head>
<body>
    <!-- ШАПКА САЙТА -->
    <header>
        <h1>Tech Insights</h1>
        <p>Глубокий анализ технологий будущего</p>
    </header>
    
    <!-- ГЛАВНОЕ МЕНЮ -->
    <nav>
        <ul>
            <li><a href="#home">Главная</a></li>
            <li><a href="#articles">Статьи</a></li>
            <li><a href="#about">О нас</a></li>
        </ul>
    </nav>
    
    <!-- ОСНОВНОЕ СОДЕРЖАНИЕ -->
    <main>
        <!-- САМОСТОЯТЕЛЬНАЯ СТАТЬЯ -->
        <article>
            <h2>Революция искусственного интеллекта</h2>
            <p>Содержание статьи...</p>
        </article>
        
        <!-- ДОПОЛНИТЕЛЬНАЯ ИНФОРМАЦИЯ -->
        <aside>
            <h3>Популярное</h3>
            <p>Другие интересные материалы...</p>
        </aside>
    </main>
    
    <!-- ПОДВАЛ САЙТА -->
    <footer>
        <p>© 2024 Tech Insights. Все права защищены.</p>
    </footer>
</body>
</html>

Давайте разберем каждый семантический тег по порядку.

Основные семантические теги и их назначение

1. <header> - Шапка (верхний колонтитул)

Это не просто «верх страницы». <header> представляет вводный контент, обычно группу вводных или навигационных средств.

Где используется:

  • В начале <body> - как шапка всего сайта (логотип, слоган)
  • Внутри <article> или <section> - как заголовок конкретного раздела
<!-- Шапка всего сайта -->
<header>
    <img src="logo.png" alt="Логотип">
    <h1>Мой учебный портал</h1>
</header>

<!-- Шапка внутри статьи -->
<article>
    <header>
        <h2>Как выучить HTML за неделю</h2>
        <p>Автор: Иван Разработчиков | Дата: <time>2024-03-15</time></p>
    </header>
    <p>Текст статьи...</p>
</article>

2. <nav> - Навигационный раздел

Представляет блок навигационных ссылок. Важно: Не все группы ссылок должны быть внутри <nav> ! Только основные блоки навигации (главное меню, оглавление статьи).

<!-- Правильно: главное меню -->
<nav aria-label="Основное меню">
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/courses">Курсы</a></li>
        <li><a href="/blog">Блог</a></li>
        <li><a href="/contacts">Контакты</a></li>
    </ul>
</nav>

<!-- НЕ нужно оборачивать в <nav>: -->
<div>
    <a href="#top">Наверх</a> | 
    <a href="privacy.html">Конфиденциальность</a> <!-- Это не основная навигация -->
</div>

Атрибут aria-label - дополнительно описывает назначение навигации для скринридеров.

3. <main>- Основное содержимое

Самый главный контент страницы. На странице должен быть только один <main> ! Это то уникальное содержание, ради которого пользователь пришел на страницу.

<body>
    <header>...</header>
    <nav>...</nav>
    
    <!-- ВСЕ ОСНОВНОЕ ЗДЕСЬ -->
    <main>
        <h1>Заголовок основной статьи</h1>
        <p>Самая важная информация страницы...</p>
    </main>
    
    <footer>...</footer>
</body>

4. <article> - Независимый контент

Представляет самостоятельную, законченную единицу контента, которую можно взять и использовать отдельно от всего сайта.

Что может быть <article>:

  • Пост в блоге
  • Новость
  • Комментарий на форуме
  • Виджет
  • Любой контент, который имеет смысл сам по себе
<article>
    <h2>10 причин изучать веб-разработку в 2024</h2>
    <p>Веб-разработка продолжает быть одной из самых востребованных профессий...</p>
    
    <footer>
        <p>Теги: <mark>карьера</mark>, <mark>обучение</mark></p>
    </footer>
</article>

5. <section> - Логический раздел

Группирует тематический контент. Обычно начинается с заголовка (<h2> - <h6>).

<article>
    <h1>Руководство по HTML5</h1>
    
    <section>
        <h2>Введение</h2>
        <p>HTML5 — это пятая версия языка разметки...</p>
    </section>
    
    <section>
        <h2>Семантические элементы</h2>
        <p>Семантика позволяет придать смысл разным частям страницы...</p>
    </section>
</article>

Важное правило: Если вы просто хотите обернуть контент для стилизации - используйте <div>. Если контент логически завершен и тематически сгруппирован - используйте <section>.

6. <aside> - Боковая панель, дополнительная информация

Контент, который косвенно связан с основным содержимым. Часто используется для:

  • Боковых панелей
  • Списков связанных статей
  • Блоков «Читайте также»
  • Рекламных баннеров
  • Биографии автора
<main>
    <article>
        <h1>Как работает CSS Grid</h1>
        <p>Основной контент статьи...</p>
    </article>
    
    <!-- Дополнительная информация -->
    <aside>
        <h3>Материалы по теме</h3>
        <ul>
            <li><a href="#">Flexbox за 5 минут</a></li>
            <li><a href="#">Адаптивный дизайн</a></li>
        </ul>
    </aside>
</main>

7. <footer> - Подвал (нижний колонтитул)

Завершающий блок. Содержит информацию об авторе, контакты, ссылки на связанные документы, копирайт.

Как и <header>, может использоваться:

  • В конце <body> - подвал всего сайта
  • Внутри <article> и <section> - подвал конкретного раздела
<!-- Подвал всего сайта -->
<footer>
    <p>© 2024 Веб-академия. Все права защищены.</p>
    <address>
        Email: <a href="mailto:info@webacademy.ru">info@webacademy.ru</a>
    </address>
</footer>

<!-- Подвал статьи -->
<article>
    <h2>Заголовок</h2>
    <p>Текст...</p>
    
    <footer>
        <p>Опубликовано: 15 марта 2024</p>
        <p>Автор: <a href="/author/ivan">Иван</a></p>
    </footer>
</article>

Дополнительные семантические теги

<figure> и <figcaption> - Иллюстрация с подписью

<figure>
    <img src="code-screenshot.png" alt="Пример HTML-кода">
    <figcaption>Рис. 1: Пример семантической разметки статьи</figcaption>
</figure>

<time> - Дата и время

<p>Опубликовано: <time datetime="2024-03-15T14:30">15 марта 2024, 14:30</time></p>

Атрибут datetime - машинно-читаемый формат для парсеров.

<mark> - Выделенный текст

<p>Важно: <mark>семантические теги не влияют на внешний вид</mark>, только на смысл.</p>

Сравнение подходов к структуре в HTML

Теперь, когда мы обсудили что из себя представляет семантика и какие семантические теги используются, давайте сравним как раньше выглядел код на HTML4 и как выглядит сейчас.

Как было раньше (HTML4):

<div id="header">
    <div class="logo">Мой сайт</div>
</div>

<div id="nav">
    <div class="menu">Главная | О нас</div>
</div>

<div id="content">
    <div class="post">
        <div class="title">Новость дня</div>
        <div class="text">Текст новости...</div>
    </div>
</div>

<div id="sidebar">
    <div class="related">Похожие статьи</div>
</div>

<div id="footer">
    <div class="copyright">© 2024</div>
</div>

Проблемы этого подхода:

  1. Браузер видит только «кучу непонятных div»
  2. Скринридер не понимает структуру
  3. Поисковики не могут определить, где важный контент
  4. Код плохо читается

Как выглядит сейчас (HTML5 с семантикой):

<header>
    <h1>Мой сайт</h1>
</header>

<nav>
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
    </ul>
</nav>

<main>
    <article>
        <h2>Новость дня</h2>
        <p>Текст новости...</p>
    </article>
    
    <aside>
        <h3>Похожие статьи</h3>
        <!-- Дополнительный контент -->
    </aside>
</main>

<footer>
    <p>© 2024</p>
</footer>

Преимущества нового подхода:

  1. Читаемость кода - сразу понятно, где что находится
  2. Лучшее SEO - поисковики ценят семантическую разметку
  3. Доступность - скринридеры правильно озвучивают структуру
  4. Удобство стилизации - можно стилизовать по тегам
  5. Будущая совместимость - современный стандарт

Практика: Преобразование div-верстки в семантическую

Задача: Преобразуйте «дивную» верстку в семантическую HTML5

<div class="page-wrapper">
    <div class="top-section">
        <div class="site-name">Блог о путешествиях</div>
        <div class="site-description">Истории из поездок по миру</div>
    </div>
    
    <div class="menu-container">
        <a href="#" class="menu-item">Главная</a>
        <a href="#" class="menu-item">Фото</a>
        <a href="#" class="menu-item">Контакты</a>
    </div>
    
    <div class="content-area">
        <div class="story-block">
            <div class="story-title">Мое путешествие в Японию</div>
            <div class="story-date">10 августа 2025</div>
            <div class="story-text">Текст статьи о поездке...</div>
            
            <div class="photo-container">
                <img src="japan.jpg" class="story-photo">
                <div class="photo-caption">Гора Фудзи на рассвете</div>
            </div>
        </div>
    </div>
    
    <div class="side-panel">
        <div class="panel-title">Другие путешествия</div>
        <div class="related-story">Италия 2024</div>
        <div class="related-story">Норвегия 2025</div>
    </div>
    
    <div class="bottom-info">
        <div class="legal-text">© Все права защищены 2026</div>
        <div class="contact-email">Email: travel@blog.ru</div>
    </div>
</div>

Подсказки для решения:

  1. Каждый <div> с классом замените на соответствующий семантический тег
  2. Для меню используйте <nav> и список <ul>
  3. Для статьи - <article>
  4. Для фотографии с подписью - <figure> и <figcaption>
  5. Для даты - <time>
  6. Для боковой панели - <aside>

Шаг за шагом: Создаем семантическую страницу

Шаг 1: Создайте базовую структуру

Создайте файл semantic.html и скопируйте базовый каркас:

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Мой семантический сайт</title>
</head>
<body>
    <!-- Здесь будет ваша структура -->
</body>
</html>

Шаг 2: Добавьте основные семантические блоки

Внутри <body> добавьте:

<header>
    <h1>Веб-дневник начинающего разработчика</h1>
    <p>Мой путь в изучении фронтенда</p>
</header>

<nav>
    <ul>
        <li><a href="#home">Дневник</a></li>
        <li><a href="#projects">Проекты</a></li>
        <li><a href="#resources">Ресурсы</a></li>
    </ul>
</nav>

<main>
    <!-- Основной контент будет здесь -->
</main>

<footer>
    <p>© 2024. Создано с любовью и семантикой.</p>
</footer>

Шаг 3: Наполните <main> контентом

<main>
    <article>
        <header>
            <h2>День 5: Открытие семантики HTML5</h2>
            <p>Дата: <time datetime="2025-10-15">15 октября 2025</time></p>
        </header>
        
        <section>
            <h3>Что я узнал сегодня</h3>
            <p>Сегодня я открыл для себя семантические теги HTML5...</p>
            <mark>Важное открытие: семантика не меняет внешний вид, но меняет смысл!</mark>
        </section>
        
        <section>
            <h3>Пример кода</h3>
            <figure>
                <pre><code>&lt;article&gt;
    &lt;h2&gt;Заголовок&lt;/h2&gt;
    &lt;p&gt;Содержание...&lt;/p&gt;
&lt;/article&gt;</code></pre>
                <figcaption>Правильная разметка статьи</figcaption>
            </figure>
        </section>
        
        <footer>
            <p>Теги: <strong>HTML5</strong>, <strong>семантика</strong>, <strong>обучение</strong></p>
        </footer>
    </article>
    
    <aside>
        <h3>Недавние записи</h3>
        <ul>
            <li><a href="#">День 4: Блочные и строчные элементы</a></li>
            <li><a href="#">День 3: Работа с формами и таблицами</a></li>
        </ul>
    </aside>
</main>

Шаг 4: Откройте в браузере

Сохраните файл и откройте его в браузере. Попробуйте:

  1. Добавить еще одну статью <article>
  2. Внутри статьи создать несколько <section>
  3. Добавить изображение с <figure> и <figcaption>
  4. Использовать тег <time> для разных дат

Что вы узнали

  1. Семантика - это использование тегов в соответствии с их смыслом
  2. 7 основных семантических тегов: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>
  3. Правила использования:
    • <main> только один на странице
    • <article> для независимого контента
    • <section> для логических разделов с заголовком
  4. Дополнительные теги: <figure>, <figcaption>, <time>, <mark>
  5. Преимущества семантики:
    • Лучшее SEO (поисковая оптимизация)
    • Улучшенная доступность
    • Читаемый и поддерживаемый код
    • Современный стандарт веб-разработки
